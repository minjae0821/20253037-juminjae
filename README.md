# !리눅스 명령어 중에서 top, ps, jobs, kill에 대해 조사해보자!

## ps란?
**ps**는 명령 실행 시점의 프로세스 목록을 스냅샷으로 보여주는 역할을 합니다.

### 특징
- 실시간이 아니라 정적 정보 제공  
- 옵션 조합이 강력하여 스크립트에서 자주 사용  
- grep과 함께 특정 프로세스 검색에 용이  

### 자주 사용하는 명령어
```
ps aux                  # 시스템 전체 프로세스 조회
ps -ef                  # 풀 포맷으로 프로세스 출력
ps aux | grep nginx     # 특정 프로세스 검색
ps -u 사용자명           # 특정 사용자의 프로세스 보기
```

## top란?
**top**은 CPU/메모리 사용량과 시스템 부하를 실시간으로 갱신하여 보여주며, 인터랙티브하게 정렬과 필터링을 바꿀 수 있습니다.

### 특징
- CPU/메모리 사용량 실시간 확인 
- 기본 3초 주기로 갱신  
- 인터페이스 내에서 프로세스 제어 가능 

### 자주 사용하는 명령어
```
P                  # CPU 사용률 기준 정렬
M                  # 메모리 사용률 기준 정렬
k                  # 해당 프로세스를 종료
d                  # 갱신 주기 변경
```

### 명령어 실행 예
```
top
top -p 1234         # 특정 PID만 추적
top -u user         # 특정 사용자 프로세스만 보기
```

## jobs란?
**jobs**는 현재 셸 세션에서 시작한 작업들을 관리합니다. 시스템 전체가 아니라, 지금 사용하는 셸에 한정된 백그라운드/중지된 작업 상태를 빠르게 확인할 수 있습니다.

### 출력 상태 
- Running : 실행 중
- Stopped : 일시 중지
- Done : 완료

### 자주 사용하는 명령어
```
jobs	                  # 현재 작업 목록 표시
fg                      # %작업번호	해당 작업을 포그라운드로 복귀
bg                      # %작업번호	백그라운드에서 실행 재개
jobs -l	                # PID 포함하여 출력
```
### 명령어 실행 예
```
sleep 100 &
jobs
fg %1
bg %1
```

## kill이란?
**kill**은 PID 또는 작업 번호에 신호를 보내어 중지, 종료, 재시작 등의 동작을 유도합니다.

### 자주 사용하는 신호
| 번호 | 신호 | 설명 |
|:----:|:---------:|:-----------------------------:|
| 15 | SIGTERM | 정상 종료 요청 |
| 9 | SIGKILL | 강제 종료 |
| 2 | SIGINT | Ctrl + C |
| 18 | SIGCONT | 중지된 프로세스 재개 |
| 19 | SIGSTOP | 프로세스 중지 |

## 명령어 개요 정리 테이블
| 명령어 | 핵슴 역할 | 권장 사용 시점 | 
|:-:|:-:|:-:|
| ps | 프로세스 스냅샷 조회 |특정 프로세스 탐색/스크립트용 |
| top | 실시간 시스템/프로세스 모니터링 | 즉시 상태 파악/정렬/종료 필요 시 |
| jobs | 현재 셸의 백그라운드 작업 조회 | 포그라운드/백그라운드 전환 관리 |
| kill | 시그널로 프로세스 제어 | 안전 종료→강제 종료 순서 권장 |
